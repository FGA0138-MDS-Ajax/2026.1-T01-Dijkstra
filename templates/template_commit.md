# Template de Commit

## Estrutura Usada
<tipo>: <descrição curta>

## Tipos de Commit

- feat: nova funcionalidade
- fix: correção de bug
- documentation: alteração na documentação
- style: formatação, espaços, ponto e vírgula, etc.
- refactor: refatoração sem alteração de funcionalidade
- test: adição ou modificação de testes
- chore: tarefas de manutenção/configuração

---

## Exemplos

### Nova funcionalidade
feat: adiciona sistema de autenticação

### Correção de bug
fix: corrige erro de cálculo na média

### Documentação
documentation: atualiza guia de instalação

### Refatoração
refactor: reorganiza estrutura de componentes

### Testes
test: adiciona testes do módulo de login

### Manutenção
chore: atualiza dependências do projeto

---

## Boas práticas

- Use frases curtas e objetivas
- Escreva no imperativo
- Evite commits genéricos como:
  - update
  - mudanças
  - ajustes
  - commit final
